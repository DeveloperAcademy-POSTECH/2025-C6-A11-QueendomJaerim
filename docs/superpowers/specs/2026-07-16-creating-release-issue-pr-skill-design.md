# 릴리즈 이슈·PR 생성 스킬 설계

## 목표

이 저장소에서 릴리즈 이슈와 `develop → main` PR을 안전하고 일관된 형식으로 생성하는 프로젝트 로컬 스킬을 제공한다.

## 구조

- 스킬 위치: `.agents/skills/creating-release-issue-and-pr`
- 구성 파일: `SKILL.md`, `agents/openai.yaml`
- 외부 변경을 일으키는 자동화 스크립트는 포함하지 않는다.

## 동작

1. 실제 기본 브랜치가 `develop`인지 확인하고 작업 트리가 깨끗한지 검사한다.
2. 릴리즈 버전을 사용자에게 확인하며 임의로 결정하지 않는다.
3. 동일 버전의 열린 이슈와 PR이 있는지 먼저 조회해 중복 생성을 막는다.
4. `develop`을 체크아웃하고 `origin/develop`을 fast-forward pull한다.
5. `origin/main...develop` 변경 범위를 확인한다.
6. 저장소 릴리즈 템플릿에 맞는 이슈를 생성한다. `release` 라벨이 없으면 라벨 없이 생성한다.
7. 생성한 이슈를 연결한 `develop → main` 릴리즈 PR을 생성한다.
8. 생성 결과와 이슈 코멘트 상태를 검증한다.
9. 릴리즈 후 `release/vX.Y.Z`가 `develop`에 포함됐는지 확인하고, 미포함 시 백머지 PR을 생성한다.

## 안전 정책

- `/start vX.Y.Z` 이슈 코멘트는 사용자가 해당 턴에 명시적으로 요청하지 않으면 작성하지 않는다.
- 릴리즈 PR을 머지하거나 태그를 생성하지 않는다.
- 기존 브랜치나 변경 사항을 삭제하지 않는다.
- 이미 직접 백머지된 릴리즈에는 빈 PR이나 no-op 커밋을 만들지 않는다.

## 베이스라인 실패

- 스킬 없이 수행했을 때 다음 패치 버전을 사용자 확인 전에 추정했다.
- 이슈 템플릿에 선언됐지만 저장소에는 존재하지 않는 `release` 라벨을 사용해 이슈 생성이 실패했다.
- `/start` 코멘트가 PR을 우회해 직접 병합하는 워크플로우임을 별도 조사해야 했다.

## 검증

- 공식 `quick_validate.py`로 스킬 구조와 frontmatter를 검증한다.
- 필수 문구와 안전 정책을 정적 검사한다.
- 실제 외부 시스템을 수정하는 forward test는 수행하지 않는다.
