# Release Skill

## 목적

`creating-release-issue` 스킬은 QueenCam 릴리즈 전에 지정 양식의 GitHub 릴리즈 이슈만 생성한다.

## 사용법

릴리즈 버전을 명시해 호출한다.

```text
$creating-release-issue v1.1.9 릴리즈 이슈 만들어줘
```

자연어로 요청해도 된다.

```text
v1.1.9 릴리즈 이슈를 지정 양식으로 만들어줘
```

버전이 없으면 스킬이 생성 전에 버전을 질문한다. 동일 버전 이슈가 이미 있으면 새 이슈를 만들지 않고 기존 이슈 URL을 반환한다.

## 생성 내용

- 제목: `🎤 [Release] vX.Y.Z`
- 본문: 개요, 버전, 체크리스트, 릴리즈 일자, 노션링크, 비고, 참조자
- 체크리스트: 릴리즈 범위 확정 항목은 완료, `/start` 코멘트 항목은 미완료
- 라벨: 저장소에 정확한 `release` 라벨이 있을 때만 추가

## 범위 밖 작업

스킬은 다음 작업을 수행하지 않는다.

- `/start vX.Y.Z` 코멘트 작성
- 릴리즈 또는 백머지 PR 생성
- release 브랜치 생성
- `main` 또는 `develop` 병합
- 버전·빌드 번호나 CHANGELOG 수정
- 태그 생성 및 푸시

이슈에 `/start vX.Y.Z` 코멘트를 남기면 `.github/workflows/release.yml`이 위 릴리즈 작업을 수행한다. 워크플로우 실행은 릴리즈 이슈 생성과 분리해 명시적으로 요청한다.
