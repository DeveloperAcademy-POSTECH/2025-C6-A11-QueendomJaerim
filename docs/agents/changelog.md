# Changelog

## CHANGELOG.md 갱신

- 작업이 완료되면 반드시 `CHANGELOG.md`의 `[Unreleased]` 섹션에 작업 내용을 정리한다.
- 변경 성격에 맞춰 `Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, `Security` 중 하나에 bullet을 추가한다.
- 사용자에게 보이는 기능 변경, 내부 정책 변경, 문서 변경 모두 누락하지 않는다.
- 아직 릴리즈되지 않은 작업은 새 버전 섹션을 만들지 않고 `[Unreleased]`에 둔다.

예시: [CHANGELOG.md](../../CHANGELOG.md)

```markdown
## [Unreleased]

### Added

- 에이전트 작업 가이드 문서 추가

### Changed

- 에이전트 작업 정책을 토픽별 문서로 분리
```
